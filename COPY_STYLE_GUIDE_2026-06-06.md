# Triplanio — гайд по копирайту и голосу (en / es / ru)

**Дата:** 2026-06-06 · **Репо:** `triplanio_new` (source of truth) · **Метод:** ux-copy + ручная сверка `src/lib/i18n/locales/*` со сверкой по функционалу (компоненты-потребители).
**Контекст:** продолжение P0 voice-пасса от 2026-06-05 ([[triplanio-design-ux-audit]], [[triplanio-localization]]). Этот документ — **спецификация**, по которой правится копирайт, и источник для раздела «Голос продукта» в Notion.

Цель прохода (по запросу Pavel): убрать из видимого текста (1) неестественные для клиентского приложения слова, (2) технические термины из разработки, (3) «роботизированность», (4) тексты, не соответствующие реальному функционалу. Плюс: EN/ES регистр, вынос захардкоженных строк в i18n.

## 1. Голос

- **RU — на «ты»** (неформально, дружелюбно, спокойно). Императив: «Добавь», «Выбери», «Укажи». Канон уже зафиксирован; этот проход добивает протечки «вы» (`Перетащите`, `загрузите`, `Ваши`, `Вас пригласили` → `тебя`).
- **EN** — friendly, plain, second person («you»). Sentence case по умолчанию.
- **ES** — на «tú» (informal), sentence case.
- Тон: помощник-консьерж, а не система. Коротко, по-человечески, без канцелярита и без бодрого маркетингового «!». Одна мысль — одна строка.

## 2. Терминология (канон)

| Сущность | RU | EN | ES |
|---|---|---|---|
| Поездка целиком | **путешествие** (контекстно допустима «поездка», где звучит естественнее — реш. Pavel 2026-06-06) | trip | viaje |
| Город в маршруте | город | city | ciudad |
| Переезд между городами | переезд (трансфер — только если транспорт) | transfer | traslado |
| Событие | событие | event | evento |
| Жильё | отель / жильё | hotel / stay | hotel / alojamiento |
| Pro-доступ | Pro | Pro | Pro |

«Трип», «поездка» (как замена путешествию вне естественных мест), «taб», «энтити» — **только в коде/комментариях**, не в UI.

## 3. Запрещённые в UI слова → замена (МЕХАНИЧЕСКИЕ, применять)

Это технический/девелоперский язык, протёкший в интерфейс. Замены однозначны.

| Найдено | Язык | → Замена |
|---|---|---|
| метаданные | ru | детали / данные (выровнять к EN «details», ES «detalles») |
| идентификатор (из) сессии | ru/en/es | переформулировать ошибку по-человечески: «Сессия истекла. Войди снова.» |
| парсинг / parsing | ru | распознавание (брони/документа) |
| драфт (транслит) | ru | черновик |
| апгрейд / Pro-апгрейд | ru | переход на Pro / Подключить Pro (translit недопустим) |
| инбокс | ru | Входящие / Уведомления |
| режим структуры | ru | режим редактирования |
| тогл/чекбокс/конфиг/инстанс/рендер | любой | по смыслу: переключатель / настройка / … (если найдётся в UI) |
| `- ` в начале строки-подсказки | все | убрать ведущий дефис |
| `✓ ` / `⭐ ` / `• ` / завершающие `→`/`←` в тексте | все | убрать; галочки/иконки — через `<Icon>` |
| `$45/мес` и любые `$`-суммы в тексте | все | убрать сумму или локализовать (партнёрская цена — см. §6) |

**Направленные стрелки `{from} → {to}`** в переездах/маршрутах — оставить (семантические).

## 4. Регистр

- **EN:** sentence case для кнопок, лейблов, заголовков секций, подсказок («Trip settings», «Calendar view», «Hotel selection», «Edit details»). Title Case оставить только для настоящих имён-брендов фич, если так задумано (обсуждается). ALL-CAPS — только через CSS `text-transform`, не в строке.
- **ES:** sentence case везде (известный долг [[triplanio-es-titlecase-debt]]); первое слово и имена собственные с заглавной, остальное строчными. Бел��й список проперов: Pro, Telegram, Stripe, PDF, названия городов/стран.
- **RU:** sentence case (русский и так не любит Title Case).

## 5. КОНТЕНЦИОЗНОЕ — НЕ менять, вынести на утверждение Pavel

Это переименования фич/разделов/шагов и тон маркетинга — продуктовые решения. Субагенты их **только фиксируют**, не правят.

- **Семейство «аддон / add-on / complemento»** — целевое слово (дополнение? возможность? расширение? extra? feature?) и заголовок секции `trip.section_addons` («Enhancements & Add-ons» / «Аддоны» / «Mejoras y complementos»). Сейчас в ES разнобой: «complemento» vs «módulo».
- **Названия шагов планнера:** `planner.step_cities` («Скелет/Trip skeleton/Esqueleto») и `planner.step_review` («Финальный драфт/Final draft/Borrador final»). Рекомендация: «Города и даты» / «Проверка» (или «Маршрут» / «Проверка»).
- **Названия «линз»/разделов трипа**, если меняются как продуктовые имена.
- **Тон маркетинговых блоков:** `sub.js` (питч Pro), пустые состояния `trips.js`, `account.cancelled_note`.
- **Кнопка «Апгрейд путешествия» → «Подключить Pro»** — если меняет смысл лейбла, подтвердить (translit всё равно убрать).

## 6. Функционал ↔ текст (сверка по §, complaint #4)

Каждую правку сверять с компонентом-потребителем: текст должен описывать то, что фича реально делает. Особое внимание:
- кнопки/тосты, обещающие действие (а делают другое);
- подсказки/hint про ограничения (лимиты, Pro-гейты) — проверять по реальной логике гейта;
- партнёрские/ценовые строки (`service.*`, `sub.*`) — цифры из Stripe/партнёра, не хардкод.

## 7. Технические инварианты (не сломать)

- **Не переименовывать ключи** — только значения (ключи завязаны на `t()`).
- Сохранять плейсхолдеры `{x}` и плюрал-семьи `_one/_few/_many`.
- Не трогать экранирование/кавычки-апострофы (`\'`), не ломать JS-синтаксис файла.
- Число ключей в en/es/ru должно совпадать после прохода (0 потерянных).
- Бот/уведомления (`telegram`, `notif`) по решению Pavel в этот проход правим как обычный UI (они в тех же i18n-файлах), но фактический текст бота, идущий из edge-функций, — отдельно (не здесь).
