ВСЕГДА перед тем как сказать «готово» по UI-правке — прогоняй reuse/unification-аудит: ищи существующий компонент / класс / правило / токен / брейкпоинт / хук / паттерн (grep) и ПЕРЕИСПОЛЬЗУЙ его; новое вводи ТОЛЬКО с явным обоснованием, почему ни одно существующее не подходит; никогда не создавай вторую систему, перекрывающую существующую (брейкпоинт/токен/класс/компонент) — выравнивайся к соседнему коду; найденные свои же дубли — схлопывай. Заканчивай UI-работу строкой «Reuse audit: reused …; new: none | <name + обоснование>». Это закреплено как Hard rule #6 в repo `CLAUDE.md` (triplanio_new) и axis (c) в Definition of Done.

**Why:** повторяющийся провал, который Pavel ловит ПОСЛЕ тестов: ввёл новый брейкпоинт 640px вместо привязки к уже существующим 880/760 → «мёртвая зона» 641–880 (хедер мобильный по 880, bottom-nav только ≤640); плодил новые back-классы/стили вместо переиспользования. Должно ловиться ДО завершения, а не по тычку.

**How to apply:** до завершения любой UI-правки — grep существующих аналогов, переиспользовать, выровнять к соседнему коду; новое — только с обоснованием в ответе; в конце дать строку «Reuse audit». Брейкпоинты в коде уже заданы (≈880/760/640) — не плодить новые. Связано: [[feedback-design-for-scale-not-now]], [[triplanio-style-token-audit]], [[triplanio-ui-audit]].

**★Усиление 2026-06-19 (Pavel, по экранам create-flow):** (1) **APPROVAL-гейт на ВСЁ новое** — новый токен/CSS-класс/общий компонент/брейкпоинт/переключатель/кнопка/любой кастом вводится ТОЛЬКО после явного согласования Pavel в плане; обоснования недостаточно — сначала спроси. Дефолт: «тот же элемент с адаптивным вариантом», а не «написать новый». (2) **i18n-дедуп — часть reuse-аудита:** один ключ = один namespace-файл; ключ в двух файлах (напр. `ai_plan.*` и в `trip.js`, и в `ai_plan.js` с разными значениями — последний спред в `locales/<lang>/index.js` молча побеждает) = баг; при правке строк экрана grep ключа по en/es/ru, схлопнуть к каноничному, отрапортовать. (3) Один логический элемент = ОДИН компонент с вариантами/состояниями (никаких бледных копий); пример долга: `TripStartControl` (planner) ≈ дубль `startDateControl` (TripStructureEdit) — оба рисуют `.ts-startctl`. Закреплено в repo CLAUDE.md (rule #4 дедуп, rule #6 approval-гейт).
