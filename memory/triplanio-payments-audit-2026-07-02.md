Двойная оплата закрыта (предчек 409 включая past_due → портал; кросс-продуктовая гонка = duplicate+needs_review+Sentry by design). Бесплатного Pro путей не найдено; гранты/RLS платёжного контура образцовые (0 грантов на энтайтлмент-колонки, recompute только service_role — сверено live prod+dev).

**P0 (подтверждено live): каталог читает ДРОПНУТУЮ колонку `provider_price.provider_price_id`.** Дроп в `20260629190000_trip32_drop_dead_payment_columns.sql:20`; TRIP-155 (`catalog.ts:67` select, `:121` update) ре-ввёл её в код, а миграция `20260701211537` колонку НЕ вернула. **P1 (риск клиента, единственный не-самолечащийся путь): pro_trip теряет право при crash между insert purchase и recompute.** `stripe-webhook/index.ts` ветка checkout.session.completed: идемпотентный гард «строка purchase с этим provider_ref уже есть → `break`» БЕЗ recomputeTrip → Stripe-ретрай после падения recomputeTrip находит строку и помечает processed, `is_pro_trip` не выставлен никогда. Компенсаций нет: `reconcileTripEntitlement` лечит только stuck-PRO (зовётся при is_pro_trip=true), stuck-FREE для pro_trip нет; FE `StripeReturnModals` для kind=trip НЕ поллит («вебхук успевает до редиректа»). Фикс: recomputeTrip(trip_id) перед `break` (+желательно FE-поллинг checkSubscriptionStatus по kind=trip, симметрично sub).

**P2:** (1) checkout.session.completed с кривой/чужой metadata (Payment Link, ручной тест) = тихий no-op → processed без reportPaymentAnomaly — нужен `checkout_unresolved`-алерт; (2) статус `incomplete_expired` не входит в CHECK `subscription_status_check` → ensureWrite throw → событие вечно failed (шум, права не искажает) — маппить в 'expired' или расширить CHECK; (3) `provider_customer` без UNIQUE(user_id,provider) + `getProviderCustomerId` берёт старейшую строку → множественные cus_ ломают reconcile (класс старого «cancel не понизил Pro»); (4) `uq_subscription_provider_sub` потерял `provider` из ключа (arbiter upsert) — латентно до второго провайдера; (5) side-effects не самоистекают: право самоистекает (is_user_pro end>now), но аддоны/telegram гасятся только webhook/reconcile-on-read; `get_pending_reminders` НЕ гейтит is_trip_pro → потерянный deleted-вебхук + неактивный юзер = бот работает бессрочно (нужен ночной sweep или is_trip_pro-гейт в бот-путях); (6) TRUNCATE/TRIGGER/MAINTAIN-гранты anon+authenticated на trips/users (TRUNCATE обходит RLS; PostgREST не экспонирует — латентно) + DELETE на users (users_delete_own) в обход anonymize_my_account; (7) EventEditDialog — форк trip-Pro-резолва сырым invoke мимо useTripProStatus (нужен isOwner — расширить хук).

**P3:** рефанд одиночного инвойса живой подписки → мигающее право (следующий invoice.paid вернёт active) — нужна политика; выигранный диспут не восстанавливает Pro + reconcile ре-revoke'ает (charge.disputed остаётся true после победы); checkSubscriptionStatus не проверяет участие → probing isPro/isOwner любого tripId; origin-чек checkout/portal строже CORS (превью Vercel получают 400); Pro.jsx не обрабатывает `TRIP_ALREADY_PRO` (409 → сырой английский); таймаут поллинга 20с гасит awaitingWebhook при ещё-Free вердикте (деньги защищены idem-key, UX нет); TripLimitDialog fail-open на клиенте (сервер закрывает); гонка лимита Free в enforce_trip_limit (2 параллельных create → 2 трипа, копеечно); webhook_event без retention + полный PII-payload.
