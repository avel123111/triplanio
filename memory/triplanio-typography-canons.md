★РЕАЛИЗОВАНО на ветке (PR #331 в dev) 2026-07-02 — TRIP-165 «унификация токенов». Весь текст приложения сведён к **10 каноническим текст-стилям**; правка одного канона меняет стиль везде, хвостов нет (гард BLOCKING).

**Гарнитуры — единая суперсемья, self-host:** `Golos Text` (текст И заголовки, заменил Rubik+Nunito) + `JetBrains Mono` (коды/идентификаторы). Оба OFL, лежат в `public/fonts/{golos,jetbrains}/*.woff2` (woff2 по unicode-range latin/latin-ext/cyrillic/cyrillic-ext), `@font-face` в `src/design/fonts.css` (импортится из `index.css`). 4 старых загрузчика Google Fonts удалены; `index.html` preload'ит основные веса Golos. Токены `--font-display`/`--font-ui`/`--font-sans`/`--font-body` → Golos, `--font-mono` → JetBrains Mono. Ре-вендоринг шрифтов — заново из Google css2 с Chrome-UA, не руками.

**10 канонов** (определены в `src/design/app.css`, блок «ТИПОГРАФИКА»): `.t-display` 54/700, `.t-title` 40/700, `.t-heading` 26/600, `.t-subheading` 19/600, `.t-label` 16/600 (всё Golos-display); `.t-body` 14/400/lh1.5 (читаемый текст), `.t-ui` 14/600 (лейблы/значения/эмфаза), `.t-meta` 12.5/600 (даты/вторичное/хинты), `.t-micro` 11/700/UPPERCASE/трекинг (теги/бейджи/эйбрау); `.t-mono` 12.5 JetBrains (коды). Каждый класс ФИКСИРУЕТ гарнитуру+размер+вес+интерлиньяж+трекинг. **Цвет в каноны НЕ входит** — отдельная ось (цвет-токены/`.muted`). Веса схлопнуты к 3 (400/600/700), интерлиньяж/трекинг — к канонам.

**+2 канона (TRIP-165 аудит 2026-07-02, PR #338 в dev, одобрены Павлом):** `.t-nano` 10px/600 (плотные НЕ-капс подписи — субтайтлы/каунтеры booking-карточек) и `.t-caption` 11px/600 (малые НЕ-капс подписи). Заведены потому что набор 10 не покрывал реальную шкалу плотных booking/editor-карточек: размеров **10px и 15px** среди канонов не было, а **11px-без-капса** конфликтовал с обязательным UPPERCASE у `.t-micro`. Итого канонов **12**. `fs-strong` (15px, своего канона нет) собран на `.t-label` (16px, +1px). Капс-эйбрау 10px = `.t-nano` + сохранённый в компоненте `text-transform:uppercase` (гард его не трогает).

**Механизм привязки — co-selector (единый источник, без правок JSX):** общие CSS-классы вписаны прямо в правило канона как доп-селекторы, напр. `.t-ui, .btn, .field__label, .mbrow__name, … { … }`; локальная типографика из класса удалена (остались раскладка+цвет). Так правка одного правила канона меняет утилиту, компонент И все «сырые» использования разом (`.btn` ×53, `.field__label` ×18 и т.п.). Инлайновые элементы без класса получают `.t-*` прямо на элементе. Глобальные `h1–h4` и `.eyebrow` тоже co-selector'ены в каноны. ~320 компонентных классов app.css + page-CSS (Budget/Docs/Calendar/…) + острова (login/public/join/terms/privacy) + лендинг — все на канонах. Маркетинговые бесповые шрифты лендинга (Inter Tight/Instrument Serif/Space Grotesk) убраны → Golos; clamp-герои → канон-токены `var(--fs-*)` с адаптивом ступеньками по `@media` (без сырого clamp/px).

**Закрытый набор = 10 канонов + орто-модификаторы (комбинируются с любым каноном, единственные легальные носители веса/lh вне канонов):** `.t-strong` (font-weight:700 — инлайн-эмфаза «то же, но жирнее», ось `<strong>`), `.t-flush` (line-height:1 — флеш-центровка крупных цифр/глифов), `.num` (tabular-nums), цвет (`.muted`/токены). Оба модификатора привязывают членов через co-selector (`.t-strong, .app-side__item.active, .lkv .v, … {}`). Правило: если по ходу «нужен 3-й модификатор» — это сигнал, что элемент просится в канон, а не в новый модификатор. Любой текст = 1 канон [+ модификатор из этого закрытого списка].

**Гард (`scripts/check-design-tokens.mjs`): `TYPO_COMP_ENFORCED=true` (BLOCKING).** ★TRIP-165 аудит 2026-07-02: **снят `isCss`-гейт** — ярусы `font-size:Npx`/`font-weight`/`line-height` теперь сканируют и `<style>`-блоки внутри `.jsx` (CSS-in-JS), раньше это была слепая зона (booking-компоненты Stay22/Viator/Fork + TripStructureEdit прятали ~44 ручных font-size/weight в `<style>` мимо канонов — теперь мигрированы co-selector'ом). Регексы `font-size:`/`font-weight:`/`line-height:` матчат CSS-синтаксис (дефис) → не ловят инлайновый camelCase. Также ловится инлайн `fontSize:'var(--fs-*)'` (allowlist `TYPO_INLINE_VAR_ALLOW` = AppErrorBoundary+LandingPage), и per-line `design-token-exempt` добавлен в ярус weight/lh (layout line-height стопки цены). Валит CI на: (1) сыром `font-size:Npx`/`clamp()`/`Nem` вне TYPO_WHITELIST (`app.css` СНЯТ с whitelist → энфорсится; остались только index/login с preflight-ресетами) — сканирует `src/` + `public/landing.css`; (2) инлайновом `fontSize/fontWeight/lineHeight/letterSpacing/fontFamily` в компоненте (composition); (3) **сыром `font-weight`/`line-height` в компонентных/страничных CSS И JSX-`<style>`** — легально только в `WEIGHT_LH_ALLOW` (app.css=дом канонов+модификаторов+база `body`; index/login/fonts.css/landing.css/PublicTrip.css=base-root'ы standalone-страниц). Escape-hatch: аннотация `design-token-exempt` в строке. Exempt по природе: краш-экран `AppErrorBoundary` (без токенов намеренно), аватар-инициалы (глиф от размера контейнера), inline-эмфаза чата/`@mention` (`emphasize()`/`mentionStyle` — ось `<strong>`: наследует Golos+размер, свой только вес). `@media` anti-iOS-zoom (16px инпуты) и адаптивные ступеньки на токенах — не нарушения.

**Ре-аудит «всё ли на канонах» (2026-07-02, эта ветка):** прошёл по слепым зонам гарда (не по «проходит ли он» — он BLOCKING, всегда зелёный на смердженном). Итог: практически весь UI-текст на канонах; Tailwind именованных `text-sm/xs/…` нет (только в CSS-комментах истории), все `var(--fs-*)` резолвятся, инлайн-типо = только exempt-острова (AppErrorBoundary/LandingPage/avatar-глифы). **Единственная находка — трекинг капс-эйбрау мимо канона:** гард фиксировал size/weight/lh, но НЕ сканировал `letter-spacing` (а трекинг — ось канона). 3 booking-эйбрау (`.va-from`/`.s22-price span`/`.fork-partner__mid span`) держали в компонентном `<style>` `text-transform:uppercase; letter-spacing:.04em`, переопределяя канон `.t-nano` (`letter-spacing:0`) по каскаду. **Фикс (одобрено Павлом):** капс-оверлей (uppercase+.04em) сведён в ОДИН co-selector в app.css (сразу после t-nano-правила), дубли из 3 компонентов убраны; `.s22-stars{letter-spacing:.5px}` (разрядка глифов ★) помечен `design-token-exempt`. **Гард усилен:** добавлен ярус `letterSpacingNum` (allow-дом = `WEIGHT_LH_ALLOW`: app.css/index/login/landing/fonts/PublicTrip) — любой сырой `letter-spacing` вне канон-дома валит CI (негатив-тест подтверждён). Осталось известным микро-островом (не UI-текст приложения): `font-family` гард в сыром CSS не сканирует — живые не-`inherit`/не-`@font-face` использования только в standalone/маркетинг-островах (`JoinTrip.jsx` литерал `'Golos Text'`, `landing.css` декор `.collage__card--notes` ui-monospace).
