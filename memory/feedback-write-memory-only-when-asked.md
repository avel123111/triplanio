Записывать что-либо в память (`memory/`) **только если Илья явно об этом попросил**. Не создавать и не обновлять memory-файлы проактивно, «на всякий случай» или по собственной инициативе.

**Почему:** 2026-07-03 (TRIP-184) я сам, без просьбы, завёл `triplanio-launch-offer.md` по ходу задачи. Илья это скорректировал: решает он, что достойно памяти. Проактивная запись засоряет базу знаний и фиксирует то, что он не считает нужным хранить.

**Как применять:** выполняя задачу, держи выводы в ответе/деливери (файлы, тред Linear, PR), а в `memory/` пиши только по явной команде «запиши в память» (или равнозначной). Сомневаешься — не пиши, максимум спроси. Исключение — вот эта самая заметка: заведена по прямой просьбе.
