★TRIP-189 (PR в dev 2026-07-04): все карточно-панельные поверхности сведены к
ЕДИНОМУ источнику - пять токенов в `src/design/app.css` и ровно две роли-скина.

Токены (`:root` = светлая, `:root[data-theme="dark"]` = тёмная, JS не участвует,
всё декларативно через `data-theme` на `<html>`):
- `--r-xl: 20px` - единый радиус всех поверхностей (был разнобой 24/20/16);
- `--surface-card` - фон: светлая `rgba(255,255,255,.72)` (слегка прозрачный,
  чтобы blur давал матовое стекло и в свете), тёмная `rgba(255,255,255,.07)`
  (стекло поверх `--bg` #0c0e1c);
- `--surface-card-border` - бордер: светлая `rgba(45,55,115,.22)`, тёмная
  `rgba(255,255,255,.17)`;
- `--shadow-glass: 0 18px 50px -26px rgba(0,0,0,.5)` - тень только у роли glass;
- `--glass-blur: 16px` - backdrop-blur только у роли glass.

Две роли (групповые селекторы в app.css, единственное место со скином):
- `.surface-glass` (+ `.statbar`) - «плавающая»: тень + backdrop-blur. Полоса
  Города/Страны и поверхности на `/trips`.
- `.surface-panel` (+ `.card`, `.statcard`, `.wdg`, `.panel`) - «плоская»: без
  тени и блюра. Панели/виджеты на экране трипа (Overview/Бюджет/Участники).

Правила поддержки: примитивы НЕ дублируют скин - подключаются к роли через
групповой селектор, а в своём блоке держат только внутреннюю раскладку
(падинги/хедеры/overflow). Новую поверхность вешать на `.surface-glass` или
`.surface-panel`, не переопределять фон/бордер/радиус руками. Глобальные
`--surface`/`--line` НЕ трогали - инпуты/меню/чипы/строки остались как были
(иначе «остеклился» бы весь UI в тёмной теме). Карточки-постеры трипов `.tc`
(фон-картинка + скрим) вне ролей: выровнен только радиус на `--r-xl`, тень/фон
свои. Имена `.tp-glass-card`/`.tp-panel`/`--card-border`/`applyLook()` из тикета -
из HTML-прототипа `Triplanio.dc.html`, в реальном React/Vite-репо их нет.
Усиливает [[feedback-reuse-first-unification]] и [[triplanio-typography-canons]].
