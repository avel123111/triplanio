Второй, продуктовый аудит Triplanio (2026-06-05), надстройка над [[triplanio-ui-audit]] (тот закрыл код-слой: токены/типошкала/дедуп/движки). Отчёт: `DESIGN_UX_AUDIT_2026-06-05.md` (корень репо). Интерактивный концепт визуала: `DESIGN_VISION_CONCEPT.html` (на боевых токенах, тогглы light/dark + Atlantic/Editorial).

**3 главных вывода:**
1. 🔴 **Сломан голос продукта (копи).** «ты»/«вы» воюют в пределах экрана (trips.js, sub.js, planner.js); грамматически битые строки («Ты уверены?», «Откуда ты вылетаете?», «ваш город»); 3 названия сущности (путешествие/поездка/трип); хардкод `$`-цен в копи при мультивалютном Stripe-чекауте; дубли-синонимы ключей + опечатка «рукамми». Всё чинится в `lib/i18n/locales/` без кода = P0, дёшево/максимум заметности.
2. 🔴 **★Consolidation-пасс** (он же ★ хэндоффа) — токены унифицированы, но booking-row/card/chip/линзы пересобраны в каждом экране → дрейф. Эталон линзы = `CalendarLens`. Это корень «разноголосицы».
3. 🟡 **Визуал корректный, но не фирменный.** Видение «editorial concierge»: включить спящие рычаги системы (serif-дисплей Instrument Serif в витринных местах, геро-обложки, цветные корешки `--ev-*`, единый AI-сюрфейс, 8-pt ритм, один primary на экран). Бренд-синий и токен-контракт НЕ трогаем.

Дорожная карта P0(копи)→P1(consolidation+движки+дисплейная шкала)→P2(UX ядра)→P3(a11y-хвост: 19 confirm/alert, 136 title=)→P4(визуал). Текущие метрики (grep 2026-06-05): 44 инлайн fontSize≥16, 144 Btn vs 32 Button+44 ui-импорта, 19 onMouseEnter.

**Поправки Pavel к видению (важно):** (1) длинное тире «—» — это AI-тэлл, НЕ переходить на него; задача обратная — вычистить тире-«клей» там, где по логике нужна запятая/точка/двоеточие; (2) editorial/tech `data-vibe` ВЫПИЛИТЬ — одна дизайн-система без вариаций (serif не вводим); (3) CalendarLens — НЕ эталон day-divider (там его нет, это сетка-календарь); day-divider надо проектировать.

**P0 копи ✅ СДЕЛАНО 2026-06-05** (354 правки, RU+EN+ES, 102 файла валидны, 0 потерянных ключей): RU→«ты» (все вы-формы убраны), де-AI тире во всех языках (0 `" - "`-клея и `—`), «трип» убран, опечатка «рукамми», `$` из sub.js убран, эмодзи/капс/стрелки из строк, ES-грамматика. Скрипты: outputs/p0_ru*.py, p0_en.py, p0_es.py. **Осталось:** решение `поездка`→`путешествие` (~30 RU строк, спорно); дубликат-ключ `trips.ai_desc` (удалить мёртвый); ES Title Case (отд. трек [[triplanio-es-titlecase-debt]]).

Прим.: app.css `--fs-display` токены были pre-existing uncommitted (наследие хэндоффа). Осталось P1: ★consolidation + свод 16 shadcn-примитивов + дедуп условных ховеров — все визуальные, ждут живого QA.

Чат = опц.фича, выкл по умолчанию. **ВЫВОД (на полной базе): приложение реально хорошо сделано и консистентно на ВСЕХ поверхностях; «разноголосица» из код-аудита — код-уровень (bespoke), НЕ видимая юзеру; большой рерайт визуала НЕ оправдан.** «Красивее» = (а) характер/элевация в витринных моментах (ФОТО-обложки вместо градиентов = дешёвый вау, микро-восторг), (б) короткий список реальных багов: формат дат(чиню), валюты Pro($/€), 4 CTA в геро, бюджет «/0₽» при пустом плане, плюрал «1 активных»/«4 городов», статус «Оплачено» plain→бейдж, планнер шаг-1 пустая карта. ⚠️ для Pro-линз нужен ВЛАДЕЛЬЦЕВ Pro-трип (на зрительском половина закрыта).

**★★ СТРУКТУРНЫЙ ДИАГНОЗ (2026-06-05, Pavel дважды отчитал за поверхностность — оценивал поэлементно вместо структуры; подключил frontend-design/emil/frontend-ui-eng/web-quality/design-critique):** настоящая проблема — на уровне СТРУКТУРЫ приложения, не элементов. **Нет единого скелета — почти каждый экран свой layout-архетип.** (1) Нет единой оболочки: в трипе лейбл-рейл 220px+крошка; редактор collapsed icon-рейл 56px+своя топ-панель; аккаунт-уровень без рейла центрованный — 3 рамки. (2) Каждая линза переизобретает content-layout: Хронология геро+лента+рейл 320px; Бюджет стат+мастер-детейл без рейла; Календарь полноширинная сетка; Карта 60/40; Чат чат+рейл 280px; Участники/Настройки центр узкий ~900px. Правый рейл то 320, то 280, то маршрут, то нет. (3) Три paradigm панели детали одной сущности: модалка (лента-событие) / левая in-place панель (редактор-город) / инлайн-карточка (CityHero). (4) Два языка: таблица (только редактор) vs карточки. (5) Generic-база: Inter + градиент-обложки как единственная графика (фото нет) + плоские карточки = «AI/шаблон» по frontend-design. **ФИКС = большой дизайн-рефакторинг: одна оболочка + одна content-сетка (фикс max-width + слот рейла постоянной ширины) + одна система панелей (рекоменд. right/in-place везде, убить модалки) + фото-обложки (CityPhoto) вместо градиентов.** Полный отчёт — секция «★★ ГЛАВНОЕ: структурный диагноз» в DESIGN_UX_AUDIT. Связь [[triplanio-editor-panels-redesign]] (уже указывал на in-place панели). Чат = опц.фича (включил на Copy of Spain, чтобы увидеть): чат-карточка+правый рейл участники/@Triplanio.
