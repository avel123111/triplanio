## ПРОТОКОЛ: Анализ ПЕРЕД кодом (без исключений)

При любой задаче по `triplanio_new` — СНАЧАЛА читаю, ПОТОМ пишу.

### Шаг 1 — Найти все релевантные файлы base44
Для каждого UI-элемента (кнопка, диалог, варнинг, виджет):
- Найти его в `triplanio_base44/src/` — компонент, страница, хелпер
- Прочитать файл ПОЛНОСТЬЮ, не беглым просмотром
- Выписать: что делает каждый handler, какие диалоги открывает, какие функции вызывает

### Шаг 2 — Пройти всю цепочку зависимостей
Каждый импорт и вызов — читать до конца:
- `onAddHotel(visit)` → найти где это вызывается → `setHotelChoice(...)` → `BookingChoiceDialog` → `HotelDialog`
- `onAddTransfer(from, to)` → `setTransferEdit(...)` → `TransferDialog`
- `sortVisits(visits)` → читать `validation.js` → понять `kind: start/end/transit`
- Не останавливаться на первом файле

### Шаг 3 — Проверить дизайн
- Открыть `/triplanio_new/src/pages/redesign/Screen*.jsx` соответствующий экран
- Сверить каждый визуальный элемент: цвета, иконки, шрифты, плейсхолдеры, состояния

### Шаг 4 — Написать план и показать Pavel
ПЕРЕД написанием кода — написать:
1. Что нашёл в base44 (файлы, логика каждого элемента)
2. Что буду делать (какие компоненты создам, какие диалоги портирую)
3. Что неясно (явно задать вопросы)

Pavel подтверждает план → только тогда пишу код.

### Что НЕ делать
- НЕ угадывать логику если не нашёл её в коде
- НЕ делать `window.__navigate?.('X')` вместо открытия диалога
- НЕ хардкодить текст — использовать те же русские строки что в base44 локалях (i18n ключи портировать)
- НЕ начинать кодить после прочтения одного файла

**Why:** Трижды сделал ошибки из-за того что не дочитал base44 до конца:
1. Кнопка "Добавить переезд" → navigate вместо TransferDialog
2. Кнопка "Добавить отель" → navigate в HotelsLens вместо HotelDialog
3. Кнопка "Редактировать" → navigate в Settings вместо isEditMode
4. Варнинги трансфера → не учёл kind:start/end/transit из validation.js

**How to apply:** Каждый раз когда Pavel даёт задачу — ДО кода пишу summary "вот что я нашёл в base44, вот мой план". Если задача большая — разбиваю на части и согласовываю каждую.
