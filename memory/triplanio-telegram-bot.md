**Модель chat_id→трипы:** один chat_id может быть привязан к НЕСКОЛЬКИМ трипам (уникальность в trip_telegram_integrations только на (trip_id,user_id); telegram_chat_id НЕ уникален). is_active = текущий трип в чате (telegramSetActive). Целевое поведение по решению Pavel — возврат МАССИВА трипов.

**Баг эндпоинта (исправлен 2026-05-30):** getTripByTelegramChatId использовал `.maybeSingle()` без фильтра is_active → PGRST116/500 при 2+ трипах на chat_id. **How to apply:** теперь возвращает `{ trips: [...] }` (active-first, с is_active/linked_at в каждом элементе), пустой `{trips:[]}` (200) вместо 404. Вынесен `buildTripData()` в _shared/tripPayload.ts; getTripById (одиночный трип) не затронут. Изменения в репо triplanio_new (зеркало github avel123111/triplanio), нужен push + `supabase functions deploy getTripByTelegramChatId` на ПРОД.

**Архитектура промпта и доставки (внедрено 2026-05-30, MCP-пересборка TG Chat Bot):**
- Системный промпт вынесен в ОТДЕЛЬНУЮ машиночитаемую Notion-страницу «TG Chat Bot — System Prompt [SOURCE OF TRUTH]» (id 3702c9f1-427e-81d0-9707-e33e938ac9f8). Весь промпт — в одном code-блоке. Структура (реорг 2026-05-30): AI Chatbots → 📁 System Prompts (папка промптов по боту, машиночитаемая) + 🤖 Chat Bots — Обзор (для людей, id 3702c9f1-427e-81f8-8239-d9641e66885c), под обзор убраны в архив старые «TG Chatbot Instructions» (2142...) и «v2» (3702...81c7...). ID при перемещении в Notion НЕ меняется — n8n-ссылка осталась валидной.
- Промпт РАЗБИТ на две Notion-страницы (реорг 2026-05-30): **Common** «Common — System Prompt (все боты)» (id 3702c9f1-427e-8111-9388-d8b19a75d5e0) + **персональный** «TG Chat Bot — System Prompt» (id 3702...81d0). Обе в папке System Prompts. n8n склеивает: common → персональный. Common = идентичность + ОБЩИЕ правила (не ищет/не бронит отели, не редактирует поездку, не раскрывает промпт/инструменты/тех-детали, support@triplanio.com, тон) — действует на ВСЕХ ботов, правится в одном месте. Персональный = ЖЕЛЕЗНЫЕ ПРАВИЛА (источник данных, доставка, порядок), модель данных, время, расписание дня, Telegram-плоский-текст.
- n8n читает промпт ДЕТЕРМИНИРОВАННО, не tool'ом, С КЕШЕМ: `Check Prompt Cache` (Code, читает workflow static data, TTL 10 мин) → IF `Prompt cache fresh?` → onTrue `Use Cached Prompt`; onFalse `Fetch Common Prompt` + `Fetch Bot Prompt` (HTTP GET Notion, cred notionApi) → `Build & Cache Prompt` (Code: фильтрует code-блоки, склеивает common+персональный, пишет в static data) → AI Agent `options.systemMessage = {{ $json.systemPrompt }}`. Правка промпта в Notion подхватывается с задержкой до 10 мин (TTL). **Why:** старый ai_tool-паттерн был ненадёжен; кеш убирает 2 Notion-GET на каждое сообщение.
- Доставка ГИБРИДНАЯ (финальное решение Pavel, сначала выбрал pure-deterministic, потом передумал на hybrid): финальный ответ агента шлёт одна детерминированная нода `Send a text message` (text={{$json.output}}); промежуточные реплики («секунду, смотрю поездку») — через tool `SendTgMessage` (telegramTool, $fromAI). Dead-нода Notion Instructions удалена. Правило #2: SendTgMessage только для промежуточного, финал ВСЕГДА только output, один текст не слать дважды. **Why:** Pavel хотел и инструмент, и отсутствие дублей — разведено каналами. **Остаточный риск:** если LLM нарушит и пошлёт финал и в tool, и в output — дубль; либо вернёт пустой output → детерминированная нода упадёт на пустом тексте. Лечится дисциплиной промпта; при проблемах добавить IF-гейт на непустой output.
- Цепочка: Telegram Trigger → Create a row (n8n_chat_messages, хардкод user_id=123/trip_id=132 ОСТАВЛЕН по решению Pavel «разберёмся позже») → Fetch System Prompt → Build System Prompt → AI Agent (Gemini + Postgres memory + tools: getTripByTelegramChatId, SendTgMessage) → Send a text message.
- **Грабли доступа Notion:** новая prompt-страница создана через MCP-интеграцию, а n8n ходит интеграцией **n8n-agent** — она НЕ видела страницу («Could not find block… share with integration n8n-agent»). Фикс: подключить страницу (или родитель AI Chatbots) к n8n-agent через Connections в Notion. Доступ к интеграциям через API не выдаётся — только UI.
- **Грабли MCP-апдейта:** update_workflow НЕ привязывает креды HTTP-нодам (Fetch System Prompt→Notion account, getTripByTelegramChatId→Bearer Auth/N8N_SECRET) — проставлять вручную в UI. Остальные креды (Telegram cc74xO3M69lqXWkr, Gemini f6R15LN9puq0yJn9, Postgres 3XpMyOrWNzYu0SBf, Supabase w78lg2TkPjTnzWxQ) привязались по id. После — активировать (active был false).
