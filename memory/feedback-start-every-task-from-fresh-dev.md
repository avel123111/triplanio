**Pavel (2026-06-28):** «любая работа — любой запрос (в активной или в новой сессии) должен начинаться с подтягивания свежего состояния dev. Всегда есть параллельные задачи и мержи, которые могут нарушить нумерацию миграций или создать конфликты, потому любая активность должна начинаться с подтягивания дев.»

**Why:** репозиторий живёт параллельно — другие задачи/агенты мёрджат в `dev` постоянно (в эту же сессию TRIP-141 за час влились #225→#226→#228 + чужой #227 TRIP-135). Работа от устаревшего `dev` = конфликты при PR, дубли версий миграций (таймстамп-коллизии ломают `db push`), правки поверх уже изменённого кода. Это прямой компаньон [[feedback-new-commits-need-new-pr-after-merge]] (там — про push ПОСЛЕ мержа; здесь — про старт ДО работы) и [[triplanio-migration-naming-drift]].

**How to apply:**

**★ СТАРТОВЫЙ ГЕЙТ (в начале КАЖДОГО запроса/задачи, до анализа и кода):**
```
git fetch origin dev
```
- **Новая ветка под задачу** — всегда от свежего: `git checkout -b cyrus1/<task>-<short> origin/dev` (НЕ от локального/устаревшего dev, НЕ от чужой ветки).
- **Продолжаешь в активной сессии/ветке** — перед новой порцией работы тоже `git fetch origin dev` и сверься: не разошлась ли база (особенно если между твоими действиями прошло время или Pavel что-то мёрджил). Если твоя ветка уже отстала и её PR ещё открыт — это нормально (merge произойдёт через PR), но при конфликтах/миграциях лучше перебазировать на свежий dev или снять правки на новую ветку от origin/dev.
- **Миграции:** перед `supabase migration new` обязательно подтяни dev и глянь последний таймстамп в `supabase/migrations/` — твой файл должен быть СТРОГО новее журнала (иначе out-of-order, см. [[triplanio-migration-naming-drift]] и [[triplanio-personal-docs-teardown]] про баг обгона таймстампа). Нумерация `00NN` запрещена — только `<YYYYMMDDHHMMSS>_*`.
- **Не доверяй локальному `origin/dev` из прошлой сессии** — он протух. Первое действие = fetch.

Короткое правило: **fetch dev → потом думай/кодь**. Никогда не начинай работу со стейта, который не переподтянул сейчас.
