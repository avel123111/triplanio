**Бизнес:** Viator (дочка Tripadvisor) = экскурсии/активности, affiliate **Basic** (бесплатно, ключ мгновенно, REST API есть). Комиссия **8–12%** за бронь, cookie **30 дней**. Деньги/возвраты/саппорт — на Viator, мы — витрина+трафик. Паттерн ровно как Stay22 (отели), только провайдер другой и уход на viator.com. На Basic доступны эндпоинты: `POST /products/search` (summary-карточки), `GET /products/{code}`, `GET /destinations`, `/products/tags`, single-availability; НЕ доступны bulk/modified-since/realtime-availability/full-reviews/booking. Авторизация `exp-api-key` + обяз. `Accept: application/json;version=2.0`. `productUrl` уже с нашей меткой (pid/mcid/medium) — НЕ модифицировать; кампания только через `campaign-value` в запросе. Контент не складировать (зовём вживую), храним только id.

**Ключевое арх-решение (с Pavel, объективная итерация):** Viator требует свой числовой `destinationId`, координат/имени мало. Вводим **единый справочник `cities`** (свой `id` PK; `name_en, country_code, lat, lng, time_zone`; `iata_code`/`viator_dest_id`/`getyourguide_id` nullable; `source`). Старый `iata_cities` (сид из TravelPayouts, только аэропортовые города, PK=iata-код) **сворачивается в cities**, iata-код → атрибут. `city_visits.iata_city_code` → `city_id` FK (вариант A — централизация, НЕ денормализация). Внешние id (place_id/osm_id) НЕ храним. **Матч ТОЛЬКО по координатам** (та же страна, ближайшая в R≈30км; имя в матче не участвует — Sevilla≠Seville, тёзки, языки); переиспользуем тело `resolve_iata_city` (миграция 0032) → `resolve_city_id`.

**Два процесса матча (не путать):** (1) батч Ф2 — `/destinations` → координатный матч к cities → существующие обогащаем viator_dest_id, новые (city-тип) создаём; (2) рантайм — триггер city_visits координатно ставит city_id, нет строки → создаёт (lazy).

**Фазы:** Ф1 фундамент (cities + миграция iata + репойнт триггера + перевод потребителей iata: флайт-линки/трансфер-панель/geo.js cityNameEn — читать из cities джойном в общий запрос визитов, без отд. раундтрипа; задевает ПЕРЕЛЁТЫ). Ф2 Viator: ключ `VIATOR_API_KEY`(prod+dev) → батч `scripts/enrich-viator-destinations.mjs` → edge `viatorActivities`(verify_jwt=true, паттерн stay22Accommodations) → FE `src/lib/viator.js`+`ViatorActivityList.jsx` (зеркала stay22) → монтаж в `ForkPartnerModal` type==='activity' у города → клик `usePartnerLogger`/`partner_clicks`(partner='viator', без изм. схемы).
