# Справочник конфликтов редактора структуры (валидатор A–E)

Источник: `src/lib/validation.js` → `computeTripValidation({ visits, hotels, activities, transfers })`.
Чистая функция, возвращает массив `{ level, code, message, cityId?, hotelId?, activityId?, transferId?, fromId?, toId? }`,
отсортированный «ошибки вперёд». Используется и редактором (панель + гейт), и (через `cityIdentity`) таймлайном.

## Общие правила
- **Жёсткий гейт:** пока список не пуст — кнопка «Сохранить» заблокирована. Деление `error`/`warn` — только для приоритета/цвета в UI; **сохранение блокирует любая запись**.
- **Сравнение дат — по календарному дню в таймзоне города** (`timezone` узла; luxon `startOf('day')`), не по точному времени.
- **Узел** = `city_visits`: `kind` ∈ `transit` (город с ночами), `waypoint` (пересадка, одна дата), `start`/`end` (якоря, даты null).
- **Порядок/соседство** считается по `(start_datetime, position)` — единый `sortVisits`.
- **Идентичность города** (`cityIdentity`): `external_city_id`, иначе `city_name|country_code` (lowercase). Используется для E1/E2.

## A. Узлы (даты города)
| Код | Уровень | Когда срабатывает | Сообщение | Как закрыть |
|---|---|---|---|---|
| `A1` | error | У `transit`/`waypoint` нет `start` или `end`. Якоря `start`/`end` пропускаются. | «У города «X» не заданы даты.» | Задать даты (правка ночей/старта; у waypoint — дата одна). |
| `A2` | error | `end` раньше `start` (по дню). | «У города «X» конец раньше начала.» | Поправить даты/ночи. |
| `A3-gap` | warn | Между соседними узлами разрыв **> 1 дня** (`gap > 1`). | «Разрыв больше дня между «A» и «B».» | Сдвинуть даты/ночи (gap 0 = стыковка, gap 1 = соседние дни — ОК). |
| `A3-overlap` | warn | Соседние узлы **наслаиваются** (`gap < 0`, B начинается до выезда из A). | ««A» и «B» наслаиваются.» | Поправить даты/порядок. |

## B. Отели (только выход за окно города; решение Pavel)
| Код | Уровень | Когда | Сообщение | Как закрыть |
|---|---|---|---|---|
| `B1` | warn | Заезд раньше дня прибытия в город. | «Заезд в «отель» раньше прибытия в X.» | Модалка → правка дат / «Подогнать под город» / удалить бронь. |
| `B2` | warn | Выезд позже дня выезда из города. | «Выезд из «отель» позже выезда из X.» | То же. |
| `B3` | error | `city_visit_id` указывает на удалённый/отсутствующий город (сирота). | «Бронь «отель» без города.» | Удалить бронь **или** вернуть город из трея «Убраны из маршрута». |

Пересечение двух отелей в одном городе **НЕ проверяется** (решение Pavel).

## C. Активности (выход за окно города)
| Код | Уровень | Когда | Сообщение |
|---|---|---|---|
| `C1` | warn | Начало раньше прибытия в город. | «X начинается раньше прибытия в …» |
| `C2` | warn | Конец позже выезда из города. | «X заканчивается позже выезда из …» |
| `C3` | error | Активность без города (сирота). | «X без города.» |

Закрытие — модалка: правка дат / «Подогнать под город» / удалить.

## D. Переезды
| Код | Уровень | Когда (по дню в tz города) | Как закрыть |
|---|---|---|---|
| `D1` | warn | Вылет **позже** дня выезда из города отправления. | Правка даты / «Подогнать под город» (день выезда) / удалить. |
| `D2` | warn | Вылет **раньше** дня выезда из города отправления. | То же. |
| `D3` | warn | Прилёт **раньше** дня въезда в город назначения. | То же (день въезда). |
| `D4` | warn | Прилёт **позже** дня въезда в город назначения. | То же. |
| `D5` | warn | Концы переезда **не соседние** узлы по `(start, position)` (между ними есть третий узел). Включает «назад во времени» (to раньше from). | Удалить переезд **или** вернуть прежний порядок городов. |
| `D6` | error | `from`/`to` указывает на удалённый узел (сирота). | Удалить переезд. |

D1/D2 = «вылет должен быть в день выезда из A», D3/D4 = «прилёт в день въезда в B».
Составной перелёт = цепочка отдельных переездов между соседними узлами через `waypoint` (на каждую пару — свой переезд).

## E. Уровень трипа (последовательность)
| Код | Уровень | Когда | Как закрыть |
|---|---|---|---|
| `E1` | warn | Между соседними по `(start, position)` узлами **разной идентичности** нет переезда `from=A,to=B`. | На коннекторе «нет переезда — добавить» → выбрать транспорт. |
| `E2` | — | Один и тот же город подряд (одинаковая идентичность) — переезд **не требуется**, E1 НЕ показывается. Это правило подавления, отдельного кода нет. | — |
| `E3` | warn | На пару соседних узлов **более одного** переезда (дубликат). | Удалить лишний переезд. |

## Заметки
- На таймлайне «нет переезда» (E1) рисуется в рендере, но идентичность городов берётся из того же `cityIdentity` — одно правило с редактором (один город подряд не варнит).
- Якоря `start`/`end` участвуют в E1 (нужен переезд старт→первый город и последний город→финиш), но не в A1/A2 (у них нет дат).
